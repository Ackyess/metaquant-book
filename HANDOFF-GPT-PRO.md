`index.html` 默认保留生活化版正文。构建脚本把精炼稿渲染成 HTML，写入 `#conciseContent` 的 JSON。读者选“精炼版”后，页面只替换前言之后各章节的 `.content`，章节结构、目录、脚注和导航仍共用一套。

选择保存在浏览器本地：

```text
mq-reading-style-v1 = story | concise
```

首次从前言进入正文，或在尚未选择时直接打开任意正文锚点，都会被二选一弹层拦住。选过以后，右上角可以重选。

## 需要重做的部分

不要继续对第二轮稿做逐句删词式 humanizer。应按章节重写精炼版，优先修复段落之间的逻辑链：

1. 每章先标出“问题 → 例子 → 概念 → 边界 → 小结”的骨架。
2. 每个概念最多保留一个真正有解释力的生活比喻，不要把比喻全部删掉。
3. 保留必要的过渡句，尤其是从故事进入定义、从定义进入风险边界的位置。
4. 不删事实、限定条件、脚注或风险提示。
5. 精炼版应比生活化版短，但不能像摘要或提纲。
6. 每改完一章，连续朗读开头六段和相邻小节的最后一段/第一段，检查是否跳步。

判断标准：读者不看标题，只按段落顺序读，也能知道“上一段为什么会走到下一段”。

## 代表性问题

生活化版常用完整铺垫：

> 真正的量化交易，不是一台预测未来的水晶球。它比较像一间开了很久的赌场。赌客关心这一把，赌场关心下一万把。

第二轮稿把结论压得更短：

> 量化交易员关心的是后者：不预言下一把，而是管理许多次下注的概率与损失。

后一句本身没错，但“后者”依赖被删薄的上下文。类似问题需要整段修，不要只补一个连接词。

## 建议接手流程

1. 先通读 `book.md` 与 `versions/03-humanizer-zh-round2.md` 的前言、第 1、2、4、7、11 章。
2. 列出每章被删掉但承担逻辑连接的句子。
3. 直接修改 `versions/03-humanizer-zh-round2.md`，不覆盖 `book.md`。
4. 每完成一章，与生活化版核对事实、数字、脚注和练习题。
5. 重新嵌入精炼稿：

   ```bash
   python scripts/build_reader_styles.py
   ```

6. 运行静态检查：

   ```bash
   python scripts/check_readers.py
   git diff --check
   ```

7. 本地预览：

   ```bash
   python -m http.server 8799
   ```

8. 浏览器至少验证：

   - 全新读者从前言进入正文；
   - 全新读者直接打开 `#04-market-microstructure`；
   - 分别选择两版后，第 1、4、7、11 章正文确实不同；
   - 刷新后保持选择；
   - 右上角可重选并切回；
   - 脚注、目录、上下章和中英切换仍可用；
   - 390×844 与桌面宽度下弹层不溢出；
   - 控制台无错误。

9. 用户确认精炼稿连贯后，再发布到现有 Cloudflare Pages 项目 `metaquant-book`。线上地址是 <https://quant.leooo.fun/>。
